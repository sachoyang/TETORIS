# TETORIS 🧱

<img width="225" height="225" alt="다운로드" src="https://github.com/user-attachments/assets/df03d9e2-ac28-44e1-adf8-dece9ce97638" />

카사네 테토 테마의 테트리스. 게임 엔진이나 프레임워크 없이 **순수 Win32 API + GDI**만으로 렌더링·입력·게임 진행을 직접 구현하고, **FMOD**로 사운드를 얹은 C++ 프로젝트입니다.

## 주요 기능

- **7종 테트로미노 + 회전** — 블록 7종 × 회전 4방향을 `int Block[7][4][4][4]` 룩업 테이블 하나로 정의. 회전은 계산이 아닌 테이블 조회(`RotateNum = (RotateNum+1) % 4`)로 처리
- **고스트 블록** — 현재 블록의 착지 위치를 회색으로 미리 표시
- **홀드 시스템** — Enter로 블록을 보관/교체. 재사용은 1회로 제한되며 사용 완료 시 슬롯에 X 표시
- **월 킥(Wall Kick)** — 벽에 붙은 상태에서 회전이 막히면 좌우 ±1, ±2칸 오프셋을 시도해 회전 보정
- **장애물 라인 상승** — `20 − Level`초마다 필드가 한 칸 올라가고 구멍 1개짜리 장애물 줄 생성. 레벨이 오를수록 주기 단축
- **레벨·점수** — 3줄 제거마다 레벨 상승(최대 10), 줄당 100점. 벽 상승 카운트다운이 3초 이하면 빨간색 경고
- **스프라이트 애니메이션** — 4프레임 시트를 `StretchBlt`로 잘라 1초마다 회전하며 춤추는 테토
- **BGM / 효과음** — FMOD를 클래스로 래핑해 배경음 루프, 효과음, 볼륨 조절 지원

## 조작법

| 키 | 동작 |
|---|---|
| ← / → | 좌우 이동 |
| ↑ | 회전 |
| ↓ | 소프트 드롭 |
| Space | 하드 드롭 |
| Enter | 홀드 / 꺼내기 |
| 메뉴 File → Start | 게임 시작 · 일시정지 |

## 구조

윈도우 **메시지 구동(message-driven)** 구조로, 별도의 게임 루프 스레드 없이 `WM_TIMER`(1초 낙하)와 키 입력 메시지가 게임을 진행시킵니다.

```
WinMain(창 생성) → 메시지 루프 → WndProc 분기
   ├─ WM_TIMER    : 블록 낙하, 장애물 타이머, 애니메이션 프레임
   ├─ WM_KEYDOWN  : 이동/회전/드롭/홀드
   ├─ WM_NewBlock : (커스텀 메시지, WM_USER+1) 새 블록 스폰 + 게임오버 판정
   └─ WM_PAINT    : 타이틀 / 플레이 / 게임오버 화면 분기
```

- **충돌 검사 단일화** — 이동·회전·낙하·고스트·게임오버 판정이 전부 `BlockCanMove(x, y)` 하나를 통과. 필드 배열 `BackGround[21][12]`의 테두리를 미리 1로 채워 벽 자체를 충돌 데이터로 사용(센티널 기법)
- **BitBlt 스크롤** — 줄 삭제·라인 상승 시 전체를 다시 그리지 않고 화면 영역을 메모리 DC로 복사 후 한 칸 이동해 붙여넣기

## 사용된 설계 기법

| 기법 | 적용 위치 |
|---|---|
| 퍼사드 + 싱글턴 + RAII | `FmodSound` — FMOD 타입을 숨기고 `AddSoundFile / EffectPlay / BGPlay` C 함수만 노출 |
| 플라이웨이트(리소스 캐시) | 사운드 경로 → ID 맵으로 중복 로드 방지 |
| 데이터 주도 설계 | 회전 룩업 테이블 — 로직을 데이터로 치환 |
| 경량 상태 기계 | `isStarted` × `PlayerState`로 타이틀/플레이/게임오버 전이 |
| 메시지 기반 이벤트 | `WM_NewBlock` 커스텀 메시지를 이벤트 버스처럼 사용 |

## 알려진 개선점 (회고)

- 메시지 루프가 `PeekMessage` busy-wait → `GetMessage` 블로킹 루프로 교체 필요
- `srand()` 미호출로 매 실행 블록 순서 동일 → 시드 추가, 나아가 7-bag 배급 방식
- `WM_PAINT` 안에서 게임오버 효과음 재생(다시 그릴 때마다 재생됨) → 상태 전이 시점으로 이동
- 전역 초기화에서 `AddSoundFile` 호출 → 정적 초기화 순서 문제, Meyers 싱글턴으로 해결
- `GetDC` 즉시 그리기 방식 → `WM_PAINT` 통일 + 더블 버퍼링
- 단일 1,900줄 파일 + 전역 상태 → `GameState` 구조체화, 로직/렌더링 분리
- 홀드·넥스트 미리보기가 현재 블록의 `RotateNum`으로 그려짐 → 기본 방향 고정
- 빌드 산출물(`x64/`) 커밋 → `.gitignore` 추가
